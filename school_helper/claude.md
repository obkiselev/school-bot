# Инструкции для Claude в рамках проекта School Helper

## 1. Порядок действий, когда пользователь просит продолжить/возобновить разработку:
- Открыть файл `PROGRESS.md` и проверить/уточнить текущий прогресс по задачам и багам.
- Продолжить по текущим приоритетам или уточнить у пользователя.
- НЕ GIT коммит на этом этапе.

## 2. Порядок действий, когда пользователь просит зафиксировать версию:
- Прочитать текущую версию из `package.json` (поле `version`) и `PROGRESS.md`
- Инкремент версии (следующий minor: 0.1.0 -> 0.1.1, без изменений мажорной)
- Обновить файлы:
  - `PROGRESS.md` — перевести текущие задачи из "В работе" в завершённые, обновить changelog.
  - `README.md` — обновить номер версии, добавить новые возможности и описание к текущей версии.
  - `package.json` — обновить поле `version`.
  - Другие файлы, в которых указана текущая версия.
- Git-операции:
  - `git add PROGRESS.md README.md package.json` (и другие обновлённые файлы)
  - `git commit -m "vX.Y: краткое описание изменений"`
  - `git tag vX.Y`
  - `git push origin main --tags`
